# Daily Scrape — July 12, 2026

## Top Story
Claude Code's token efficiency is becoming a competitive liability. Boris Cherny (Claude Code lead) just shipped `/checkup` to help users optimize their setups, but Hacker News is simultaneously surfacing that Claude Code burns 33k tokens before reading a prompt versus OpenCode's 7k—a 4.7x overhead that's sparking real frustration in the developer community. For SMBs and consultants selling AI automation, this is a cost-to-value inflection point: the "best" model isn't always the cheapest to run at scale.

## What People Are Saying
- Developers are openly souring on Claude despite loving the underlying LLM. One Hacker News thread titled "I used to love Claude, but the latest models are slowly ruining it" hit 48 points with 61 comments—signal that perceived value has shifted from capability to overhead and UX friction.
- There's growing pushback against reflexive "ask an LLM" advice. A post titled "Stop Telling Me to Ask an LLM" (189 HN points, 109 comments) suggests the hype cycle has peaked and practitioners are now filtering for when *not* to use AI—a maturity signal your clients need to hear.
- GaryVee is positioning "old school" and "human-first" approaches as underexploited business opportunities, directly countering the AI-first narrative. This suggests a market opening for consultants who frame AI integration as *augmentation* rather than replacement.

## Updates
- Boris Cherny shipped Claude Code `/checkup`—a diagnostic tool that deduplicates configs, removes unused skills/MCPs, and enables auto-mode by default. Matters because it's Anthropic acknowledging context bloat as a real problem, not a user education gap.
- Claude Code extended the May–July 2026 weekly limits promotion through the end of the month. Signals pricing/packaging is still in flux as Anthropic competes on usage caps, not just capability.
